爬虫入门总结
发表于 2020-06-06| 更新于 2020-06-08
阅读量:

浏览器请求

  • url
    • url=请求协议+网站域名+资源路径+参数
  • 浏览器请求url地址
    • 当前url对应响应+js+css+图片—>elements中内容
    • elements中内容可能与爬虫拿到的数据不同
  • 当前url所对应的响应在哪里
    • 从network中找到相应项目，点击response

HTTP与HTTPS

  • HTTP:超文本传输协议
    • 明文形式传输
    • 效率高，不安全
  • HTTPS:HTTP+SSL
    • 传输前加密，读取前解密
    • 效率低，安全
  • HTTP协议之请求
    • HTTP请求行 GET XXX HTTP/ 1.1
    • 请求头：
      • User-agent：包含浏览器信息，有些网页不好爬的话可以考虑改成手机的UA
      • Cookie：用于携带用户信息，每次请求会被带上发给对方浏览器，服务器可能根据该字段判断是否为爬虫
    • 请求体:
      • 携带数据
      • get请求没有请求体
      • post请求有请求体
  • HTTP协议之相应
    • 响应头
      • Set-Cookie:服务通过该字段设置Cookie到本地
    • 响应体
      • Url地址所对应的响应

Requests

  • 发送get，post请求

    • response = requests.get(url)
    • response = requests.post(url, data = {请求体字典})

  • response相关方法

    • response.text()
      • 如果出现乱码，先调用response.content.decode(“utf-8”)
    • response.content.decode()
      • 将响应的二进制字节流转换为str类型
    • response.request.url 发送的请求的url地址
    • response.url response响应的url地址
    • response.request.headers 请求头
    • response.headers 响应头

  • 获取网页源码的正确姿势

    • response.content.decode()
    • response.content.decode(“gbk”)
    • response.text()

  • 发送带header的请求

  • 使用超时参数

    • requests.get(url)

  • 重试模块

处理cookie的相关请求

  • 直接携带cookie请求url地址

    • 直接放在headers中
    • Cookie字典传给cookies参数

  • 先发送post请求，获取cookie, 带上cookie请求登陆后页面

    • session = requests.session() #具有的方法和requests一样
    • session.post(url, data, headers) #服务器设置在本地的Cookie会保存在session中
    • session.get(url) #会带上保存在session中的cookie，能够请求成功

json

  • json.loads
    • 将json字符串转化为python类型
    • json.loads(json字符串)
  • json.dumps({})
  • json.dumps(ret1, ensure_ascii=False, indent=2)
    • ensure_ascii:让中文显示成中文
    • indent:让输出的json不要在一行上

xpath和lxml

  • xpath

    • xpath helper插件：帮助我们确定定位的内容

  • 使用

    • /选择单层节点

    • //选择当前层级以下的所有元素

    • @的使用

      • div[@class=’???’]限定节点属性
      • /a/@href 选择a的href值

    • 获取文本

      • /a/text()

      • /a//text（）获取a下面的全部文本。

    • ./a 当前节点下的a标签
